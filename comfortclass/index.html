ComfortClass: Code Comfortably.

ComfortClass is a programming language in early development, designed by yours truly. It's object-oriented, and draws heavily from Python, which is also the language it's implemented in. ComfortClass' design is based on the concept of comfort.

What do you mean by comfort?

I'm glad you asked. Comfort can take many different forms. Here are the main ones:
  • Simplicity. ComfortClass is easy to learn.
    • It has no memory management or references.
    • The syntax is unambiguous and easy to comprehend.
    • It doesn't require explicit type declarations, and allows for very easy casting.
    • It's not verbose, but not hard to read.

  • Ease of transition. ComfortClass is similar to many major languages.
    • The syntax is C-like, similar to a large number of popular languages.
    • There are (planned) direct ports of python modules to ComfortClass.
    • It doesn't have strange restrictions that throw you through a loop.

  • Security. ComfortClass gives you total control over the features you want to use.
    • It doesn't have a full-fledged standard library.
    • All ComfortClass modules are open-source by nature.
    • It also doesn't have any tricky builtins like Python does.
    • All this allows for easy arbitrary code execution.

Can I use ComfortClass?

Not yet, unfortunately. ComfortClass and its documentation are in early development right now. However, you can see how it's doing and track the progress by going to the repository.
